# Privacy Policy — TrotroOS

_Last updated: 28 May 2026_

TrotroOS ("we", "our", "the app") connects passengers with trotro mates in Kumasi, Ghana. We collect only the data needed to make the service work.

## 1. Data we collect

| Category | What | Why |
| -------- | ---- | --- |
| **Location** | GPS coordinates **while the app is open** | Passengers see nearby mates; mates see waiting passengers on the map. |
| **Device ID** | Random ID on your phone (passengers) | Identify reservations without requiring sign-in. |
| **Identity** | Name, phone; vehicle registration (mates) | Authentication and rider safety. |
| **Reservations** | Route, seats, timestamps | Run bookings and support. |
| **Ratings** | Optional 1–5 star feedback | Help other riders choose reliable mates. |
| **Diagnostics** | Crash reports (production builds with monitoring enabled) | Fix bugs and improve stability. |

We do **not** collect background location. We do **not** sell data. We do **not** show ads.

## 2. Payments

Fares are paid **directly to your mate** in cash or mobile money when you board. In-app card/mobile-money checkout is not enabled in this version.

## 3. How long we keep data

- Live location pings stop when you leave the queue or cancel a reservation.
- Reservation records may be kept up to 12 months for disputes.
- You can clear local app cache in **Profile → Data & privacy**.

## 4. Who we share data with

- **Supabase** — database and authentication (EU-hosted infrastructure).
- **Other users** — only trip-related information (route, live position when sharing is on).
- **Law enforcement** — only when compelled by valid legal process.

## 5. Your controls

- **Profile → Privacy** — toggle location sharing.
- **Profile → Data & privacy** — export local data, clear cache, request deletion via email.
- **Android Settings → Apps → TrotroOS → Permissions** — revoke location anytime.
- **Mate sign-out** — available in Data & privacy; email support for full account deletion.

## 6. Security

All server connections use HTTPS/TLS. Database access uses Supabase Row Level Security.

## 7. Children

The service is not directed to children under 13.

## 8. Changes

We will update the date at the top when we make material changes and notify users in-app when appropriate.
